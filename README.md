# 暑假修行册 · 哪吒与敖丙

一个面向小朋友的暑假作息打卡 Web App。哪吒（火焰）与敖丙（寒冰）双卡通主题，纯本地运行，零依赖。

## 快速开始

**打开方式（任选其一）：**

1. 在文件资源管理器中双击 `index.html`，浏览器即可直接打开。
2. 用 VS Code / Cursor 的 Live Server 扩展运行，体验更流畅。
3. 想在手机上用，把整个文件夹拷贝到手机后用任意浏览器打开 `index.html`，或将这三个文件部署到任意静态托管（GitHub Pages / Vercel / Netlify）。

所有数据保存在浏览器 **localStorage**，不上传任何服务器；换浏览器或换设备需使用「导出/导入」功能迁移。

## 功能一览

### 🔥 今日 Tab
- 按时间顺序展示每日作息卡片
- 每个任务可独立打卡（点击右侧圆形按钮，火焰粒子动画反馈）
- 每个任务下方有 **完成情况备注框**，自动保存（如「读了《西游记》第三回 32 分钟」）
- 顶部实时显示当日加权星级（1–5 ⭐），满 5 星触发全屏雪花特效
- 页面底部有 **当日总体备注**，供家长留言
- 支持上一天 / 下一天 / 直接跳今天

### 📅 日期 Tab
- 按周排布日历，颜色越深表示得星越多
- 起止日期可在「模板」页自由设置；结束日期留空则一直显示到今天
- 点击任意一天，跳转到该日补打卡或查看历史

### ⚙️ 模板 Tab
- 设置孩子昵称、起止日期
- 编辑每日作息：增 / 删 / 改时间、图标 emoji、标题、权重（1–5）
- 内置一份默认作息（13 项，覆盖起床 → 就寝），首次使用即可上手
- 修改权重后所有历史日期的星级会自动重算

### 🎁 奖励 Tab
- 顶部显示「累计星星」与「可用星星」
- 内置 4 个示例奖励（冰淇淋 / 新书 / 电影 / 游乐园），可自由增删改
- 可用星星 ≥ 所需 时「兑换」按钮亮起，兑换后自动扣减并写入历史

### 底部工具栏
- **导出数据**：下载 JSON 备份（建议每周导出一次）
- **导入数据**：从 JSON 恢复
- **重置全部**：清空所有记录恢复默认（请慎用）

## 加权星级算法

```text
完成率 = Σ(已完成任务权重) / Σ(全部任务权重)

⭐⭐⭐⭐⭐  完成率 ≥ 90%
⭐⭐⭐⭐    完成率 ≥ 75%
⭐⭐⭐      完成率 ≥ 55%
⭐⭐        完成率 ≥ 30%
⭐          完成率 >  0%
```

把「晨读 / 作业 / 阅读」等关键事项的权重设到 3，把「叠被子 / 早餐」等设为 1，孩子会把精力优先放在最重要的事情上。

## 主题视觉

- 哪吒侧：火焰红 `#E63946` / 烈焰橙 `#FF7A1A` / 流光金 `#F4C542`，对应「打卡完成」「重要任务」「热情活力」
- 敖丙侧：寒冰蓝 `#3FA7D6` / 月白 `#E8F4FB` / 深海青 `#1B4E6B`，对应「日历 / 奖励 / 冷静沉稳」
- 全部头像和装饰用 **CSS + SVG** 自绘，无需任何图片素材

若想替换成真实立绘，可在工程根目录建 `assets/` 文件夹放入图片，并在 `index.html` 中把 `<svg>` 替换为 `<img src="assets/nezha.png">` 即可。

## 隐私说明

所有数据仅保存在你本地浏览器的 localStorage，不发送到任何服务器。清除浏览器数据或重装系统会丢失记录，请定期使用「导出数据」备份。

## 常见问题

**Q：换个浏览器看不到记录？**
A：localStorage 与「浏览器 + 域名」绑定。同一台电脑用 Chrome 看不到 Edge 里的数据。请用「导出 / 导入」迁移。

**Q：能不能加云同步？**
A：暂未实现。可以把整个项目部署到 GitHub Pages + 用浏览器同步，或自行接入 Firebase / Supabase。

**Q：孩子手机上没有 PC，怎么用？**
A：把文件夹拷贝到手机后用文件管理器双击 `index.html`，或部署到一个静态托管（如 Vercel）后用网址访问。
